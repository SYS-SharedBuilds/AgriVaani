# AgriVaani Live Demo Script

**Total Duration:** 3-4 Minutes
**Target Audience:** Judges for the "Build with AI: Code for Communities" hackathon.

## Prerequisites (Native Execution)
1. Ensure your `.env` contains a cloud PostgreSQL connection string (from Neon.tech or Supabase).
2. Start the Orchestration API:
   ```bash
   cd apps/orchestration-api
   npm install && npm run dev
   ```
3. Start the Recommendation Service (requires Python >= 3.9):
   ```bash
   cd apps/recommendation-service
   python -m venv venv
   # Windows: venv\Scripts\activate | Mac/Linux: source venv/bin/activate
   pip install -r requirements.txt
   uvicorn main:app --reload --port 8000
   ```
4. Start the Officer Dashboard:
   ```bash
   cd apps/officer-dashboard
   npm install && npm run dev
   ```
5. **Twilio Webhook via ngrok:**
   - In a new terminal, run: `ngrok http 3000`
   - Copy the HTTPS URL (e.g., `https://abc-123.ngrok.app`)
   - Go to your Twilio Phone Number settings -> Configure -> "A CALL COMES IN" -> set to Webhook: `<YOUR_NGROK_URL>/v1/voice/inbound`

---

### Step 1: Voice & Recommendation Engine (1 Minute)
*Goal: Show the primary farmer interface with a real phone call.*
1. State the problem: Farmers don't use smartphones; they need a voice/SMS interface.
2. Call your Twilio Trial Number from a real phone.
3. Listen to the Hindi IVR (powered by Google Cloud TTS).
4. Press '2' on the keypad for a recommendation.
5. The orchestration API hits the Recommendation Engine microservice.
6. Hear the localized rationale generated by Gemini read back over the phone call.

### Step 2: Crop Health Logging & RSK Triage (1 Minute)
*Goal: Show the multimodal AI diagnosis and human-in-the-loop escalation.*
1. Explain that a farmer sends an SMS, MMS, or voice message about a sick plant.
2. Open the "Simulate Farmer Call" (FarmerSimChat) on the dashboard. Note that this component is a scripted stand-in for the actual voice/SMS channel for demo convenience.
3. Type a message like "half my tomato plants have collapsed overnight". This hits the actual `/v1/health-cases` API. 
4. Point out that the escalation decision comes from the real Gemini diagnosis + coded rule (`needs_expert = confidence < 0.6 OR severity === "high"`), not a turn counter. Because the severity was high, it escalates to the RSK officer.
5. Click "Resolve" on the dashboard UI to show the real-time update.

### Step 3: Hardware & Alert Engine (1 Minute)
*Goal: Prove real ground-truth integration, not just LLM wrappers.*
1. Show the physical Arduino Mega 2560 setup with the pH sensor and relay board.
2. Explain the cron job `alertEngine.ts`: Plot 1 represents a deterministic dry-spell scenario (low groundwater, no rain forecast).
3. Run `python scripts/telemetry_replay.py` to trigger the ingestion logic.
4. Show the alert being generated in the database deterministically, leading to an SMS notification and relay control queue update.
